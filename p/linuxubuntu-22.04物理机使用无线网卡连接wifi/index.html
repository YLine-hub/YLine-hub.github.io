[Linux]ubuntu 22.04物理机使用无线网卡连接wifi

2026-02-13

下载 network-manager 依赖包

使用能够正常使用的 ubuntu 22.04 虚拟机

  • 在虚拟机下创建network_offline文件夹并进入
1
2

mkdir network_offline
cd network_offline

下载包

  • 使用命令下载network-manager 依赖包
1

apt download $(apt-cache depends --recurse --no-recommends --no-suggests --no-conflicts --no-breaks --no-replaces --no-enhances --no-pre-depends network-manager | grep "^\w" | sort -u)
  • 退出文件夹，并将其压缩为压缩包
1
2

cd ..
tar -czvf network_offline.tar.gz network_offline

转移network_offline包到物理机上

转移 network_offline.tar.gz

  • 拷贝net_offline.tar.gz包到u盘并插入物理机

挂载u盘

  • 创建挂载目录
1

mkdir -p /mnt/usb
  • 查看所有磁盘
1

lsblk

看到新出现的一个 sdb 磁盘，并且内部有一个 sdb1 的盘没有挂载任何东西

  • 挂载u盘
1

mount /dev/sdb1 /mnt/usb

复制 network_offline.tar.gz 到物理机

  • 创建存放的目录
1

mkdir /tools
  • 复制文件到本地
1

cp /mnt/usb/network_offline.tar.gz /tools
  • 解压目录
1
2

cd /tools
tar -zxvf network_offline.tar.gz

安装network_manager

  • 进入目录
1

cd network_offline
  • 安装
1

dpkg -i *.deb
  • 测试
1

nmcli

连接 wifi

  • 查看设备状态
1

nmcli dev status

发现显示未连接状态

  • 查看wifi列表
1

nmcli dev wifi list
  • 连接wifi
1

nmcli -a dev wifi con
  • 检查是否连接
1

nmcli

使用nmcli配置静态ip

  • 使用nmcli设置静态ip、网关和DNS
1
2
3
4

nmcli con modify "@Ruijie-403" ipv4.addresses 192.168.1.100/24
nmcli con modify "@Ruijie-403" ipv4.gateway 192.168.110.1
nmcli con modify "@Ruijie-403" ipv4.dns "8.8.8.8 8.8.4.4"
nmcli con modify "@Ruijie-403" ipv4.method manual
  • 应用设置
1

nmcli con up "@Ruijie-403"

此时，静态ip才算设置成功

  • 重启网络服务
1

systemctl restart NetworkManager
  • 验证静态 IP 是否生效
1
2
3
4

# 查看 ip
ip a show
# 查看网关
ip route show
