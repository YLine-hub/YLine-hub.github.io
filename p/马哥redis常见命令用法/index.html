INFO

显示当前节点redis运行状态信息

1
2

[root@redis-110 ~ ]# redis-cli -a 123456
127.0.0.1:6379> info

可以看到此时只有一个客户端连接

使用112服务器连接110，并查看redis节点运行状态

1
2

[root@redis-112 ~ ]# redis-cli -a 123456 -h 192.168.172.110
192.168.172.110:6379> info

可以看到，此时有两个客户端连接

使用info查询对应信息（只要info后面加上#后的内容，即可查询对应信息）

1

SELECT

默认只有16个数据库，0~15

切换数据库1

1
2
3

1
2

(error) ERR DB index is out of range

切换数据库15

1
2
3

127.0.0.1:6379[1]> select 15
OK
127.0.0.1:6379[15]>

查看数据库所有内容

1
2

127.0.0.1:6379[15]> keys *
(empty array)

添加数据

1
2

127.0.0.1:6379[15]> set a b
OK

查看数据

1
2

127.0.0.1:6379[15]> keys *
1) "a"

切换数据库0

1
2
3

127.0.0.1:6379[15]> select 0
OK
127.0.0.1:6379>

再次查看内容

禁用命令 

1
2
3

# 生产建议修改配置使用rename-command禁用一些命令
vim /etc/redis.conf
rename-command FLUSHALL ""  #flushdb和和AOF功能冲突，需要设置 appendonly no,不区分命令大小写，但和flushall （v7.2.3不冲突）

将flushall配置为随意定义的长字符串

1
2
3

[root@redis-110 ~ ]# vim /apps/redis/etc/redis.conf
......
rename-command flushall nklas4681zcaqwiehoianc654861xvdsnfoia

重启服务

1

[root@redis-110 ~ ]# systemctl restart redis

测试flushall

1
2
3
4
5
6

[root@redis-110 ~ ]# redis-cli -a 123456
Warning: Using a password with '-a' or '-u' option on the command line interface may not be safe.

(error) ERR unknown command 'flushall', with args beginning with:
127.0.0.1:6379> nklas4681zcaqwiehoianc654861xvdsnfoia
OK

可以看到flushall命令已经失效，替代而来的是nklas4681zcaqwiehoianc654861xvdsnfoia这段长字符串

禁用命令：将flushall和flushdb命令替换成空字符串

1
2
3
4

[root@redis-110 ~ ]# vim /apps/redis/etc/redis.conf
...
rename-command flushall ""
rename-command flushdb ""

之后就无法使用这两项命令

重启服务

1

[root@redis-110 ~ ]# systemctl restart redis

SHUTDOWN

关闭Redis服务,停止所有客户端连接（危险）

1
2
3

# 建议禁用此指令
vim /etc/redis.conf
rename-command shutdown ""

测试shutdown

1
2
3
4

[root@redis-110 ~ ]# redis-cli -a 123456
Warning: Using a password with '-a' or '-u' option on the command line interface may not be safe.
127.0.0.1:6379> shutdown
not connected> exit
