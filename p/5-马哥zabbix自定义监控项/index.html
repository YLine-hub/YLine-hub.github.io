key名称一般是"应用名.作用[参数]“样式存在 shell命令 与 key使用 逗号+空格（无所谓）隔开

自定义key格式

客户端可以自定义监控项，在Zabbix Agent配置文件添加内容，格式如下

  • 样式一：核心配置文件中定制
1
2
3
4
5

# cat /etc/zabbix/zabbix_agentd.conf 或者 zabbix_agent2.conf
UserParameter=<key>,<shell command>
# 支持参数
UserParameter=<key>[*],<shell command> $1 $2 .....
Include=/etc/zabbix/zabbix_agentd.d/*.conf
  • 样式二：创建独立的自定义文件
1
2
3
4

# cat /etc/zabbix/zabbix_agentd.d/*.conf 或者 /etc/zabbix/zabbix_agent2.d/*.conf
UserParameter=<key>,<shell command>
# 支持参数
UserParameter=<key>[*],<shell command> $1 $2 .....

key 细节

key 必须整个系统唯一。注意大小写是敏感的，key名允许的字符如下
0-9a-zA-Z_-.

key 使用 [*] 用于定义该key接收括号内的参数。参数须在配置监控项时给出：参数禁止使用下列字符：
\ ’ " ` * ? [ ] { } ~ $ ! & ; ( ) < >

Command: 命令用于生词key对应的值。
可以在命令中使用位置引用$1…$9来引用监控项key中的相应参数。Zabbix解析监控项key的[]中包含的参数，并相应地替换$1,…,$9。$0 会替换为完整的原始命令（在对$0,…,$9执行替换之前的命令）运行。不管位置参数（$0,…,$9）是用双引号（"）还是单引号（’）括起来，都会解析位置引用

关联item

  • 关联item
  • 在指定的Host配置界面中，为指定的host增加item属性
  • 配置item属性的时候买最好使用手工编号，因为自定义的item无法通过select选择

#!/bin/bash

# 设置环境变量
HOST="127.0.0.1"
PORT="80"
STATUS_URL="status"

# 定制错误提示信息
Usage(){
    echo "Usage: /bin/bash $0 [ active|reading|writing|accepts|handled|requests|ping ]"
}

# 判断脚本执行条件
if [ $# -ne "1" ];then
    Usage
    exit
fi

# 定制 nginx status 信息获取
curl_cmd="/usr/bin/curl -s http://$HOST:$PORT/$STATUS_URL 2>dev/null"

active(){
    ${curl_cmd} | grep "Active" | awk '{print $NF}'
}
reading(){
    ${curl_cmd} | grep "Reading" | awk '{print $2}'
}
writing(){
    ${curl_cmd} | grep "Writing" | awk '{print $4}'
}
accepts(){
    ${curl_cmd} | awk 'NR==3' | awk '{print $1}'
}
handled(){
    ${curl_cmd} | awk 'NR==3' | awk '{print $2}'
}
requests(){
    ${curl_cmd} | awk 'NR==3' | awk '{print $3}'
}
ping(){
    /usr/bin/pidof nginx | wc -l
}

# 获取数据
case $1 in
    active)
        active;;
    reading)
        reading;;
    writing)
        writing;;
    accepts)
        accepts;;
    handled)
        handled;;
    requests)
        requests;;
    ping)
        ping;;
    *)
        Usage;;
esac
