NAME
     basename, dirname – return filename or directory portion of pathname

SYNOPSIS
     basename string [suffix]
     basename [-a] [-s suffix] string [...]
     dirname string [...]

DESCRIPTION
     The basename utility deletes any prefix ending with the last slash ‘/’ character present in string (after first stripping trailing slashes), and a suffix, if given.  The suffix is not stripped if
     it is identical to the remaining characters in string.  The resulting filename is written to the standard output.  A non-existent suffix is ignored.  If -a is specified, then every argument is
     treated as a string as if basename were invoked with just one argument.  If -s is specified, then the suffix is taken as its argument, and all other arguments are treated as a string.

     The dirname utility deletes the filename portion, beginning with the last slash ‘/’ character to the end of string (after first stripping trailing slashes), and writes the result to the standard
     output.

EXIT STATUS
     The basename and dirname utilities exit 0 on success, and >0 if an error occurs.

EXAMPLES
     The following line sets the shell variable FOO to /usr/bin.

           FOO=`dirname /usr/bin/trail`
说明
  • NAME： 对命令进行一个简要的说明。
  • SYNOPSIS： 说明语法格式。
  • DESCRIPTION： 对命令进行详细说明。
  • EXIT STATUS： 说明执行完退出的情况，及返回的状态。
  • EXAMPLES： 对命令进行举例说明。

示例

NAME
       readlink - print resolved symbolic links or canonical file names

SYNOPSIS
       readlink [OPTION]... FILE...

DESCRIPTION
       Print value of a symbolic link or canonical file name

       -f, --canonicalize
              canonicalize by following every symlink in every component of the given name recursively; all but the last component must exist

       -e, --canonicalize-existing
              canonicalize by following every symlink in every component of the given name recursively, all components must exist

       -m, --canonicalize-missing
              canonicalize by following every symlink in every component of the given name recursively, without requirements on components existence

       -n, --no-newline
              do not output the trailing delimiter

       -q, --quiet,

       -s, --silent
              suppress most error messages

       -v, --verbose
              report error messages

       -z, --zero
              separate output with NUL rather than newline

       --help display this help and exit

       --version
              output version information and exit

       GNU coreutils online help: <http://www.gnu.org/software/coreutils/> Report readlink translation bugs to <http://translationproject.org/team/>

awk

一种处理文本文件的语言，是一个强大的文本分析工具。

语法 

1

awk options 'pattern {action}' file

选项参数说明：

  • options：是一些选项，用于控制 awk 的行为。
  • pattern：是用于匹配输入数据的模式。如果省略，则 awk 将对所有行进行操作。
  • {action}：是在匹配到模式的行上执行的动作。如果省略，则默认动作是打印整行。

options 参数说明：

  • -F <分隔符> 或 –field-separator=<分隔符>： 指定输入字段的分隔符，默认是空格。使用这个选项可以指定不同于默认分隔符的字段分隔符。

  • -v <变量名>=<值>： 设置 awk 内部的变量值。可以使用该选项将外部值传递给 awk 脚本中的变量。

  • -f <脚本文件>： 指定一个包含 awk 脚本的文件。这样可以在文件中编写较大的 awk 脚本，然后通过 -f 选项将其加载。

  • -V 或 –version： 显示 awk 的版本信息。

  • -h 或 –help： 显示 awk 的帮助信息，包括选项和用法示例。

local关键字

介绍

local 是一个用于声明局部变量的关键字。当在函数内部使用 local 声明变量时，该变量只能在函数内部使用，并且不会对函数外部的同名变量产生影响。这样可以确保在函数内部定义的变量不会意外地覆盖全局变量或其他函数中的变量。

#!/bin/bash

# 定义一个全局变量
global_var="全局变量"

function example_function(){
	# 定义一个局部变量
	local local_var="局部变量"
	# 内部调用局部变量
	echo "在函数内部访问局部变量：${local_var}"
	echo "在函数内部访问全局变量：${global_var}"
}


# 调用函数
example_function

# 外部调用局部变量
echo "在函数外部访问局部变量：${local_var}"
echo "在函数外部访问全局变量：${global_var}"

$和特殊变量的使用细节

单纯的$变量名

当我们定义完一个变量后（或者是已知环境变量中存在的变量名），那我们就可以通过在变量名前面加$的方式来引用这些变量

示例
  • 引用变量
1
2
3
4
5
6

#!bin/bash
field1=Hello 
field2=World

echo $field1
echo $field2

执行结果：

1
2

Hello
World

我们可以看到，我们定义的变量是可以正常打印到控制台的

使用$变量需要注意的细节

变量名后面不要直接加其他字符串，防止识别错误。

  • 在变量后直接加上字符串
1
2
3
4
5
6

#!bin/bash
field1=Hello

echo '-----------'
echo $field1World
echo '-----------'

执行结果：

1
2
3

-----------

-----------

可以发现，打印出来的结果是空。这是因为字符串和变量名中间没有空格的话，脚本在执行的过程中会把field1World看成是一整个的变量名，而这个变量名我们没有定义，所以自然是找不到结果的。

${变量名}

${变量名}和$变量名的效果差不多，作用是引用我们定义的变量（包括环境变量），但${变量名}可以显式地标识我们希望引用的具体的变量。

示例
  • 变量后面加上字符串
1
2
3
4
5
6

#!bin/bash
field1=Hello

echo '-----------'
echo ${field1}World
echo '-----------'

执行结果：

1
2
3

-----------
HelloWorld
-----------

我们可以发现，即使字符串World和变量之间不需要空格，脚本执行的时候也可以正常引用变量。

$( 脚本 )

在执行一条命令时，shell会将$()中的语句当做命令执行一遍，再将结果加入到原命令中重新执行。简单概括的话，$( 脚本 )可以让我们获取到命令执行的结果值。

示例
  • 输出当前日期
1
2
3
4
5

#!bin/bash

echo '--------------------------'
echo current date is "$(date +%Y-%m-%d)"
echo '--------------------------'

执行结果：

1
2
3

--------------------------
current date is 2025-12-04
--------------------------

我们可以看到，脚本在执行的过程中，确实是先把$()的内容执行后，再加入到了最终的文本中。需要注意的是，输出echo的内容的时候，不可以使用’’，单引号里面的内容会被当做普通文本来处理。

PS：这里稍微拓展一下，`脚本` 和$(脚本) 这两个语法实现的效果是一致的，所以有时候我们会看到有些人是用`脚本`的方式来获取脚本的执行结果的。

这两个语法主要是用来帮助我们做一些计算的。我们知道在shell脚本中，我们定义的数字默认都会被识别为字符串。所以涉及到数值运算的话，我们可以借助$(( 运算式 )) 来帮助我们完成一些计算。

和$有关的特殊变量

在shell脚本中，除了我们自定义的变量以及环境变量外，shell中还存在着一些自带的特殊变量来供我们使用。这些特殊变量在我们跑一些比较复杂的脚本的时候，给予很大的帮助。

变量 含义
$0 当前脚本的文件名
$n(n>1) 传递给脚本或函数的参数。n是一个数字，表示第几个参数。例如，第一个参数是1,第二个参数是2。若n大于10，则语法为${n}
$# 传递给脚本或函数的参数个数
$* 传递给脚本或者函数的所有参数
$@ 传递给脚本或函数的所有参数。当被双引号"“包含时，$@和@*稍有不同
$? 上个命令的退出状态，或函数的返回值。（成功为0，失败为1）
$$ 当前Shell进程ID。对于Shell脚本，就是这些脚本所在进程ID

关于$* 和$@的区别

在上一个案例中，我们会发现$*和$@打印出来的结果是一样的，那实际上这两个特殊变量有存在什么区别吗？如果这两个特殊变量没有被"“包含的时候，其实是没有区分的。但当他们被"“包含时，区别就显示出来了：$*把所有的参数看做一个整体，而$@把每个参数区分对待。

文件测试运算符与逻辑运算符

  • &&

只有前面的命令执行成功（退出状态码 0），才会执行后面的命令；如果前面命令失败，后面命令直接跳过。

  • ||

只有前面的命令执行失败（退出状态码 ≠ 0），才会执行后面的命令；如果前面命令成功，后面命令直接跳过。

#!/bin/bash

function edit_nginx_conf(){
	# 判断是否存在nginx根目录，如果不存在则创建
	[ -d /usr/local/nginx ] || mkdir -p /usr/local/nginx
	# 判断是否存在conf配置目录，如果不存在则创建
	[ -d /usr/local/nginx/conf ] || mkdir -p /usr/local/nginx/conf
	# 判读是否存在nginx配置文件，如果不存在则创建
	[ -f /usr/local/nginx/conf/nginx.conf ] || touch /usr/local/nginx/conf/nginx.conf
}

function main(){
	edit_nginx_conf
}

function log_error(){

	echo "$(date '+%Y-%m-%d %H:%M:%S')-ERROR:${msg}" >> ${APP_ROOT}/logs/$(basename $0 .sh).log
	echo "$(date '+%Y-%m-%d %H:%M:%S')-$(basename $0) is exit--------------------------------------------------------------" >> ${APP_ROOT}/logs/$(basename $0 .sh).log

	echo "$(date '+%Y-%m-%d %H:%M:%S')-$(basename $0) is exit--------------------------------------------------------------"

# 检测目录是否完整
function check_tools_directory(){

    [ -d ${APP_ROOT}/logs ] || mkdir -p ${APP_ROOT}/logs
}


# 检查nginx是否运行
function check_nginx_is_run(){
	if [ $(ps -ef | grep "nginx: master process" | wc -l) -gt 1 ]; then
		# 运行中就退出
		log_error "nginx is running... Exit the installation program "
		exit 1
	else
		log "nginx is not running..."
		check_nginx_directory_without_run
		
	fi
}

# 检查nginx目录是否存在
function check_nginx_directory(){

		log_error "The nginx directory already exists. Stop installing the program."
	else 
		log_error "The nginx directory was not found. Please uninstall it manually."
		exit 1
	fi
}

# 检查nginx目录是否存在
function check_nginx_directory_without_run(){
	if [ -d /usr/local/nginx ]; then
		# 存在就删除
		log_error "The nginx directory already exists. Stop installing the program."
	else 
		log "The nginx directory was not found."
	fi
}

# 下载安装包
function download_nginx(){
	# 查看目录下是否存在nginx安装包，不存在则下载
	[ -f ${APP_ROOT}/source/nginx-${NGINX_VERSION}.tar.gz ] || wget -P ${APP_ROOT}/source https://nginx.org/download/nginx-${NGINX_VERSION}.tar.gz
	RETVAL=$?
	if [ ${RETVAL} -eq 0 ]; then
		log "download nginx true"
	else
		log "download nginx false"
	fi
}

# 安装nginx
function install_nginx(){
	# 如果不存在目录就创建
	[ -d /usr/local/nginx ] || mkdir -p /usr/local/nginx
	# 检查是否存在安装包
	if [ -f ${APP_ROOT}/source/nginx-${NGINX_VERSION}.tar.gz ] ; then
		# 如果已经解压了
		if [ -d ${APP_ROOT}/source/nginx-${NGINX_VERSION} ] ; then
			cd ${APP_ROOT}/source/nginx-${NGINX_VERSION}
			./configure --prefix=/usr/local/nginx --with-http_stub_status_module --with-http_ssl_module
			make
			make install
			RETVAL=$?
			if [ ${RETVAL} -eq 0 ]; then
				log "install nginx true"
			else
				log "install nginx false"
			fi
		else
			tar -zxvf ${APP_ROOT}/source/nginx-${NGINX_VERSION}.tar.gz -C ${APP_ROOT}/source/
			cd ${APP_ROOT}/source/nginx-${NGINX_VERSION}
			./configure --prefix=/usr/local/nginx --with-http_stub_status_module --with-http_ssl_module
			make
			make install
			RETVAL=$?
			if [ ${RETVAL} -eq 0 ]; then
				log "install nginx true"
			else
				log "install nginx false"
			fi
		fi
	else
		log_error "No nginx installation package was found. Download stopped."
		exit 1
	fi
}

# 开放80端口
function open_80_port(){
	# 检查80 端口是否开放
	if [ `firewall-cmd --query-port=80/tcp` == "no" ]; then 
		# 开启80端口
		firewall-cmd --zone=public --add-port=80/tcp --permanent
		# 重启防火墙
		firewall-cmd --reload
		RETVAL=$?
		if [ ${RETVAL} -eq 0 ]; then
			log "open 80 port true"
		else
			log "open 80 port false"
		fi
	else 
		log "Port 80 is now open"
	fi
}

# 检测nginx服务并安装
function check_and_install_nginx_sys(){
	if [ -f /usr/lib/systemd/system/nginx.service ]; then
		log "The nginx service already exists"
	else
		check_source_nginx_sys
	fi
	sys_nginx
	RETVAL=$?
	if [ ${RETVAL} -eq 0 ]; then
		log "install nginx service true"
	else
		log "install nginx service false"
	fi
}

# 检查source目录下，是否存在nginx服务文件
function check_source_nginx_sys(){
	if [ -f ${APP_ROOT}/source/nginx.service ]; then 
		cp ${APP_ROOT}/source/nginx.service /usr/lib/systemd/system/
		chmod 744 /usr/lib/systemd/system/nginx.service
	else
		cat_nginx_service
		chmod 744 /usr/lib/systemd/system/nginx.service
	fi
}

# 写入nginx服务
function cat_nginx_service(){
# 用 cat 输出内容并写入文件(注意，EOF这段必须顶格写)
cat > "$NGINX_SERVICE_FILE" << 'EOF'
[Unit]
Description=nginx - web server
After=network.target remote-fs.target nss-lookup.target
[Service]
Type=forking
PIDFile=/usr/local/nginx/logs/nginx.pid
ExecStartPre=/usr/local/nginx/sbin/nginx -t -c /usr/local/nginx/conf/nginx.conf
ExecStart=/usr/local/nginx/sbin/nginx -c /usr/local/nginx/conf/nginx.conf
ExecReload=/usr/local/nginx/sbin/nginx -s reload
ExecStop=/usr/local/nginx/sbin/nginx -s stop
ExecQuit=/usr/local/nginx/sbin/nginx -s quit
PrivateTmp=true
[Install]
WantedBy=multi-user.target
EOF
}

# 安装成系统服务
function sys_nginx(){
	systemctl daemon-reload
	systemctl start nginx
	systemctl enable nginx
}

function main(){
    check_tools_directory
	log "$(basename $0) is running-----------------------------------------------------------"
	check_user_is_root
	check_nginx_is_run
	install_env
	download_nginx
	install_nginx
	open_80_port
	check_and_install_nginx_sys
	log "$(basename $0) is complete----------------------------------------------------------"
}

function download_mysql(){
	# 查看目录下是否存在mysql的安装包，要是不存在的话就下载
	[ -f ${APP_ROOT}/source/mysql-${MYSQL_VERSION}.tar.xz ] || wget -P ${APP_ROOT}/source https://downloads.mysql.com/archives/get/p/23/file/mysql-${MYSQL_VERSION}.tar.xz
	RETVAL=$?

		log "download mysql true"

function install_mysql(){
	# 解压文件到指定位置
	if [ -d ${APP_ROOT}/source/mysql-${MYSQL_VERSION} ]; then
		cp -r ${APP_ROOT}/source/mysql-${MYSQL_VERSION} /usr/local/mysql
	else 
		tar -xvf ${APP_ROOT}/source/mysql-${MYSQL_VERSION}.tar.xz -C ${APP_ROOT}/source/
		cp -r ${APP_ROOT}/source/mysql-${MYSQL_VERSION} /usr/local/mysql
	fi 
	# 如果没有存储数据的文件夹的话，则创建
	[ -d ${MYSQL_DATA_DIR} ] || mkdir -p ${MYSQL_DATA_DIR}
	[ -d ${MYSQL_DATA_DIR}/data ] || mkdir -p ${MYSQL_DATA_DIR}/data
	[ -d ${MYSQL_DATA_DIR}/log ] || mkdir -p ${MYSQL_DATA_DIR}/log
	if [ $? -eq 0 ]; then
		log "create mysql data directory true"
	else	
		log "create mysql data directory false"
	fi
	create_group_and_user
	# 编辑my.cnf文件
	edit_mysql_cnf
	# 安装mysql
	/usr/local/mysql/bin/mysqld --initialize --user=mysql --datadir=${MYSQL_DATA_DIR}/data --basedir=${MYSQL_DATA_DIR}
	# 获取mysql密码
	TEMP_MYSQL_PWD=$( grep 'temporary password' ${MYSQL_DATA_DIR}/log/mysql.log | awk '{print $NF}' )
	log "the mysql temporary password is ${TEMP_MYSQL_PWD}"
	# 添加mysql服务
	# add_mysql_sys
    edit_mysql_sys
	# 启动mysql
	start_mysql_sys
	# 创建mysql软链接
	link_mysql
	# 修改mysql密码
	set_root_pwd_and_remote
}

function create_group_and_user(){

	groupadd mysql
	# 创建用户 
	useradd -s /usr/sbin/nologin -r -g mysql mysql
	# 设置权限
	chown -R mysql:mysql /usr/local/mysql  
	chmod -R 755 /usr/local/mysql 
	chown -R mysql:mysql ${MYSQL_DATA_DIR}
	chmod -R 755 ${MYSQL_DATA_DIR}
	RETVAL=$?
	if [ ${RETVAL} -eq 0 ]; then
		log "create group and user true"
	else
		log "create group and user false"
	fi

	fi
	if [ $(rpm -qa | grep libaio | wc -l) -eq 0 ]; then

	fi
	if [ $(rpm -qa | grep numactl | wc -l) -eq 0 ]; then
		yum install numactl -y
	fi
	if [ $(rpm -qa | grep wget | wc -l) -eq 0 ]; then
		yum install wget -y
	fi
