zabbix 平台需求

注意：

由于 Zabbix server 需要实时与数据库处于链接状态，所以分配的物理内存越多，数据库（以及Zabbix）的工作速度就越快 Zabbix 几乎支持所有主流的 linux 系统及 windows 系统

软件配置

例如：软件 PHP 是必须的，支持版本为 8.0.0-8.4.x

  • 查看 rocky 9 php 的 yum 源版本
1

dnf list php

可以看到版本是 8.0.30

  • 查看 ubuntu 24 php 的 apt 源版本
1

apt-cache madison php

可以看到版本是 8.3+

注意：不倾向于安装 mysql 的最新版本，有可能因为版本兼容性的问题，导致后期运行出现报错

Zabbix 发版规划

目前的发布计划周期为六个月，每六个月将有一个新的Zabbix稳定版本发布。

每一年版Zabbix将会发布

Zabbix LTS（长期支持版本）发布 Zabbix LTS 版本在五年内为 Zabbix 用户提供支持服务，包括三年的全面支持（基础的、紧急的以及安全性上的问题）和两年的最底限度支持（仅限紧急的和安全性上的问题）。 Zabbix 标准版本发布。 标准 Zabbix 发行版对 Zabbix 客户提供全面支持（一般支持、关键支持、安全支持）6个月，直到下一次 Zabbix 稳定版发布，此外还将客户提供限制支持（关键支持和安全支持）6个月。

  1. 安装 zabbix 仓库
1
2

  1. 安装 mysql 服务端
1

apt install mysql-server -y

注意：如果 mysql 和 zabbix server 没有在同一台主机上，需要开放对应 ip 的地址访问能力

  1. 安装 zabbix server 和 前端工具以及代理agent
1

apt install -y zabbix-server-mysql zabbix-frontend-php zabbix-nginx-conf zabbix-sql-scripts zabbix-agent
  1. 创建数据库

由于该方法安装的 mysql 默认没有密码

1
2
3
4
5
6

$ mysql
mysql> create database zabbix character set utf8mb4 collate utf8mb4_bin;
mysql> create user zabbix@localhost identified by 'password';
mysql> grant all privileges on zabbix.* to zabbix@localhost;
mysql> set global log_bin_trust_function_creators = 1;
mysql> quit;
  1. 导入初始架构和数据

使用默认的sql文件，创建zabbix的数据库环境

1

zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql --default-character-set=utf8mb4 -uzabbix -p zabbix

对于ubuntu Desktop来说，上述命令是可以正常执行成功的，但是ubuntu server版本上可能会出现如下问题

解决方法：在 mysql 数据库中，为该用户增加 SUPER 权限即可

1

GRANT SUPER ON *.* TO zabbix@localhost;
  1. 查看数据库中的表
1

mysql -uzabbix -ppassword -e "show tables from zabbix;"

可以看到数据库中的表都已经导入了

  • 查看用户表
1

mysql -uzabbix -ppassword -e "select * from zabbix.users;"

  1. 关闭 log_bin_trust_function_creators 选项
1
2
3

$ mysql
mysql> set global log_bin_trust_function_creators = 0;
mysql> quit;
  1. 配置zabbix连接数据库

/etc/zabbix/zabbix_server.conf

1
2

$ vim /etc/zabbix/zabbix_server.conf
DBPassword=password
  1. 查看 zabbix_server.conf 中配置的行
1

grep -Env '#|^$' /etc/zabbix/zabbix_server.conf
  • -E：使用扩展正则表达式
  • -n：显示匹配的行号
  • -v：反向匹配（显示不包含模式的行）
  • '#|^$'：搜索模式
    • #：匹配注释行（以#开头）
    • |：或
    • ^$：匹配空行

  1. 查看 Nginx 软件文件
1

dpkg -L zabbix-nginx-conf

  1. 删除默认的 nginx 站点配置

/etc/nginx/sites-enabled/default，默认使用 80 端口

1

rm -f /etc/nginx/sites-enabled/default
  1. 配置 zabbix nginx 端口为 80 端口

编辑/etc/nginx/conf.d/zabbix.conf

1

vim /etc/nginx/conf.d/zabbix.conf

开启并设置端口号为 80 端口

1

listen 80;

  1. 修改 php 配置

配置文件/etc/zabbix/php-fpm.conf

1

vim /etc/zabbix/php-fpm.conf

主要修改PHP的默认时间区域为亚洲上海

1

  1. 启动 zabbix 服务
1
2

systemctl restart zabbix-server zabbix-agent nginx php8.3-fpm
systemctl enable zabbix-server zabbix-agent nginx php8.3-fpm

netstat -tnulp | egrep 're|ngi|php|zabb'

问题三

监测——主机——服务器——图形——中文不显示

该处中文字体的话不会显示

  1. 查看前端配置文件
1

dpkg -L zabbix-frontend-php | grep -E 'define|font'

  1. 查看默认的字符集
1

grep -ni _FONT /usr/share/zabbix/include/defines.inc.php

  1. 查看默认支持的字符集
1

ls /usr/share/zabbix/assets/fonts

解决方法：

到 windows 电脑上查找字符集，字符集的目录：C:\Windows\Fonts，我们选择一个字体【任何字体都可以】，选择一个 simkai.ttf，将其上传到 zabbix 的字体目录下，改名为 graphfont.ttf
