数据采集

功能软件

  • 一般情况下，数据采集的时候，主要有两种方式：
  1. HTTP 协议方式
  2. Zabbix agent + zabbix 专用协议

第二种比较好

  • zabbix server 的数据从组成方式上来说，主要靠两种采集方式：
  1. 本机采集： zabbix_agentd
  2. 跨主机组合采集：zabbix_get + zabbix_agentd zabbix_agent 收集客户端本地所有信息，守护进程 zabbix_get 收集远程客户端节点的指定的监控条目，检测环境
  • 软件简介
  1. zabbix_agentd：

zabbix_agentd 是一个专门用于在客户端节点主机上查看默认的有效监控条目信息，它本质上是一个Zabbix守护程序，用于监视各种服务器参数。 这些数据都会在 zabbix server 的最新数据界面中体现

  1. zabbix_get：

Zabbix-Get是Zabbix中的一个程序，用于Zabbix-Server到Zabbix-Agent的数据获取，通常可以用来检测验证Agent的配置是否正确。可以在启动agent之前定位错误原因。

软件信息

  • 检查zabbix-agent的软件信息
1

  • 检查zabbix-get的软件信息
1

apt info zabbix-get

  • 安装zabbix-get

属性解析：

属性解析
-s：远程Zabbix-Agent配置文件中设置的IP地址或者是主机名。如果指定的ip不存则发生报错
-p：远程Zabbix-Agent的端口。
-l：本机出去的IP地址，用于一台机器中又多个网卡的情况。
-k：获取远程Zabbix-Agent数据所使用的Key（监控条目关键字，可以通过zabbix_agentd命令获取）
