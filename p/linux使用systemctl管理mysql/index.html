jemalloc

jemalloc是一个malloc()碎片避免和可扩展的并发支持的实现。
它旨在用作系统提供的内存分配器，如FreeBSD的libc库，以及链接到C / C ++应用程序。jemalloc提供了许多超出标准分配器功能的内省，内存管理和调优功能。

安装jemalloc

  • 安装环境
1

yum install -y automake autoconf  libtool  autoreconf bzip2 git make

1

git clone https://github.com/jemalloc/jemalloc.git

[Unit]
Description=MySQL Server
Documentation=man:mysqld(8)
Documentation=http://dev.mysql.com/doc/refman/en/using-systemd.html
After=network.target
After=syslog.target

[Service]
User=mysql
Group=mysql
Type=notify
TimeoutSec=0
OOMScoreAdjust=-1000
ExecStart=/usr/local/mysql/bin/mysqld --defaults-file=/etc/my.cnf $MYSQLD_OPTS
EnvironmentFile=-/etc/sysconfig/mysql
LimitNOFILE = 65536
Restart=on-failure
RestartPreventExitStatus=1
Environment=MYSQLD_PARENT_PID=1
PrivateTmp=false

[Install]
WantedBy=multi-user.target

相关配置说明

  • TimeoutStartSec : “等待启动的时间。如果守护进程服务没有在配置的时间内发送启动完成的信号，则该服务将被认为失败， 服务将退出。”

  • TimeoutStopSec : “等待关闭的超时时间”

  • TimeoutSec : “快速配置TimeoutStartSec和TimeoutStopSec时间”。以秒为单位， “0”来禁用。默认为， 默认使用DefaultTimeoutStartSec。

  • OOMScoreAdjust: “设置进程因内存不足而被杀死的优先级。可设为 -1000(禁止被杀死) 到 1000(最先被杀死)之间的整数值”

  • EnvironmentFile: “指定配置文件，在目录前加’-’，作用是忽略文件不存在”

  • LimitNOFILE : “设置进程最大文件描述符数量的参数”

    • 软限制(Soft Limit)：进程当前实际可用的文件描述符数量
    • 硬限制(Hard Limit)：进程能够设置的最大文件描述符数量

  • LimitNOFILE = 65536

  • LimitNOFILE = 65536:65536 # 软限制:硬限制

  • RestartPreventExitStatus : “指定当服务以某些退出状态码退出时，不自动重启服务。退出状态码 1 通常表示"通用错误"或"操作不允许” "

  • Environment=MYSQLD_PARENT_PID=1 : “设置环境变量，制定父进程PID为1 (systemd)”

  • PrivateTmp : “私有临时目录”
